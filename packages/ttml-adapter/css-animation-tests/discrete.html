Discrete animation: steps(1, end) per keyframe

Each interval holds its value for the entire duration, then jumps instantaneously at the boundary — exactly as TTML calcMode="discrete".

steps(1, end): the end position means the jump happens at the end of the interval, so the incoming value is held throughout and the new value appears only when the next keyframe is reached.

index keyTime color animation-timing-function (outgoing)
0 0% red steps(1, end)

3 75% green steps(1, end)
4 100% purple (none — last keyframe)
